Caching

  1. Setup template caching.
  2. Edit index.php and include AIOMcache.php.
  3. Enable AIOM+ caching under the Caching tab. See README.md.

Minify HTML source code

Enable HTML Minify option under the Minification tab.

Minify Stylesheets or parse LESS in your template file

Minification of a single file:

<link rel="stylesheet" href="<?= AIOM::CSS('css/stylesheet.css'); ?>">

Minify multiple files into one file:

<link rel="stylesheet" href="<?= AIOM::CSS(array('css/file-1.css', 'css/file-2.less', 'css/file-3.css')); ?>">

Minify Stylesheets automatically

Enable the CSS Minify option under the Minification tab.

Minimy Javascripts in your template file

Minification of a single file:

<script src="<?= AIOM::JS('js/javascript.js'); ?>"></script>

Minify multiple files into one file:

<script src="<?= AIOM::JS(array('js/file-1.js', 'js/file-2.js', 'js/file-3.js')); ?>"></script>

Minify Javascripts automatically

Enable teh JS Minify option under the Minification tab.

.htaccess change for domain sharding

To prevent duplicate content, you need to make a change in the .htaccess file of your ProcessWire installation. Add the following at the end of the .htaccess file in the ProcessWire Webroot and change "static.mysite.com" to your subdomain and "http://www.mysite.com" to your main domain. Please take care, the %-sign at the end of the domain is required!

# ----------------------------------------------
# Domain sharding option
# ----------------------------------------------
<IfModule mod_rewrite.c>

    RewriteCond %{HTTP_HOST} ^static.mysite.com [NC]
    RewriteCond %{REQUEST_URI} !^.*\.(cur|gif|ico|jpe?g|png|svgz?|webp|eot|otf|ttc|ttf|woff|woff2|css|js)$
    RewriteRule ^(.*)$ http://www.mysite.com%{REQUEST_URI} [L,QSA]

</IfModule>
